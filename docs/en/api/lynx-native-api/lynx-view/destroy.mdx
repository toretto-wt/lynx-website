# destroy

<APISummary />

After `LynxView` becomes unavailable, you need to actively call this method to release the `LynxView` memory, otherwise there will be a memory leak.

## Syntax

### Android

```java
public void destroy();
```

### Desktop (C++)

Desktop C++ API does not expose a standalone `destroy()` method. Release the `LynxView` by destroying or resetting the owning C++ object.

```cpp
std::unique_ptr<lynx::pub::LynxView> view = builder.Build();
// ...
view.reset();
```
